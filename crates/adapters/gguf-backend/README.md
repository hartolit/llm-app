# gguf-backend

CPU GGUF inference adapter over `llama-cpp-2`.

The crate owns all llama.cpp and GGUF-specific state. It implements the portable
`domain-contracts` loader/model boundary without exposing native pointers or
vendor types. A small streaming GGUF header reader inspects admission metadata
without loading tensor data.

The adapter supports multiple logical sequences in one llama.cpp context. Native
sequence slots are allocated at sequence creation and returned through the
`LoadedModel::destroy_sequence` lifecycle hook.

The backend does not advertise `ALLOCATION_FREE_HOT_PATH`: the safe upstream
wrapper maintains internal vectors around decoded logits, and llama.cpp may
perform internal cache maintenance during execution.

The self-referential native model/context owner is generated by `self_cell` in
one private lint boundary. Handwritten adapter code remains under
`deny(unsafe_code)` and exposes no native pointer.
